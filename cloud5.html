MODULE 2 — CHAPTER 5

🖥️ IaaS & PaaS

Cloud service models, IaaS characteristics, pricing models, storage/compute/network services, PaaS benefits

⏱ ~50 min 📊 20% weightage ⭐⭐⭐ Exam Favourite
🎯 Learning Objectives: Service models overview, IaaS characteristics and when to use it, IaaS pricing models (on-demand/prepaid/auctioned), IaaS services (storage/compute/network), PaaS definition and benefits.
1.2 Cloud Service Models Overview
⭐⭐⭐ Provider vs User Responsibility:
On-Premise: User manages EVERYTHING
IaaS: Provider manages hardware/network/servers. User manages: apps, runtime, middleware, data
PaaS: Provider manages everything except apps and data
SaaS: Provider manages everything. User just uses the service.
Service TypeMajor Providers
IaaSAWS, Microsoft Azure, Google Compute Engine (GCE), DigitalOcean, Rackspace
PaaSGoogle App Engine, Apache Stratos, AWS Elastic Beanstalk, Windows Azure, Heroku, Force.com, OpenShift
SaaSGoogle Apps, Dropbox, Salesforce, Cisco WebEx, Concur, GoToMeeting
1.3 Infrastructure as a Service (IaaS)

IaaS = instead of owning hardware, you access it as a service. Think of it like renting a car instead of buying one.

IaaS Services Offered:

  • Storage Service
  • Compute Service
  • Network Service
  • Cloud Management Service

IaaS Characteristics:

  • Access through rich UI: Web Interface, Programming API, CLI
  • Pay as you Go: Charged based on time used and type of service
  • Self-service on-demand provisioning: Choose from available offerings online
  • Higher availability: Access from anywhere, any device
  • Highly Reliable: Resources pooled across data centers; auto-failover
  • Scalable: Auto-scaling feature — scale up/down based on workload

When to use IaaS:

  • When demand for computing resources is volatile
  • When new organizations want to reduce capital cost
  • When specific, trial or temporary hardware access is needed

When NOT to use IaaS:

  • When you want higher performance (on-premise is better)
  • When regulatory compliance restricts off-shoring data
IaaS Storage Services
  • Access data anytime, anywhere
  • Provider manages distributed storage devices, redundancy, and consistency
  • Issues: Internet dependency, network latency, data security
  • Pricing factors: Storage capacity, data transferred to cloud, data retrieved, reliability level
IaaS Compute Services — 3 Pricing Models

1. 🕐 On-Demand

Pay by the hour. No long-term commitment. Users can occupy/free resources on the fly. Best for users who don't know requirements in advance.

2. 💳 Prepaid (Reservation)

Pay in advance for specific time period. Cheaper than on-demand because provider can plan ahead. Best for users who know requirements in advance.

3. 🎯 Auctioned (Spot/Bid)

Users bid for resources. Bid price controlled by supply & demand. Helps providers utilize idle infrastructure. Best when resource requirement is not immediate. Useful for non-production environments. AWS calls this "Spot Instances".

IaaS Network Services

Two most widely used:

  • Load Balancing: Single access point to multiple servers. Distributes requests. Approaches: Round Robin, Weighted Round Robin, Dynamic Round Robin, Least Connections, Fastest Server
  • DNS: Hierarchical name/IP resolution. Stores IP/domain mapping for internal + external servers. Features: Geo-location DNS routing, extended security, API for developers
Cloud Management Services (CMS)
  • Cloud Value Added Management Services: Build additional features on top of existing cloud services. Automate manual tasks.
  • Cloud Integration Service (CIS): Integrates private + public clouds. Abstraction layer for multi-cloud support. Automates provisioning, scaling, monitoring, configuration.
  • Cloud Service Brokerages (CSB): Aggregates services from different providers. Organizes into searchable service catalogue.
1.4 Platform as a Service (PaaS)

PaaS = configurable application platform. Abstraction layer ABOVE hardware, OS, and virtualization stack. PaaS = IaaS + custom software stack for the application.

PaaS Benefits:

  • Write, build and deployment tools for rapid development
  • Application integration with web services and databases
  • Multi-tenancy platform for many concurrent users
  • Easy API management
  • Logging, reporting, and code instrumentation
  • Security, scalability, redundancy, auto-provisioning

PaaS Services:

  • Integration Services: Integrate data from multiple sources (on-premise, cloud, CRM, etc.). Dell Boomi = leader, integrates 40+ technologies
  • Application Development & Deployment: Supports agile/continuous delivery. Google AppEngine, Amazon Beanstalk automate build/test/release/deploy.
  • Data Analysis & Business Intelligence: Simplify data analysis, apply statistical methods for business decisions. Issues: Less control, limited customization, lack of maturity.
📋 Key Definitions
IaaS (Infrastructure as a Service)

Cloud service model providing virtualized computing infrastructure (servers, storage, networking) over the internet. Instead of owning hardware, you access it as a service.

⭐⭐⭐

PaaS (Platform as a Service)

Cloud service model providing configurable application platform (OS, runtime, middleware, dev tools) on top of IaaS. PaaS = IaaS + custom software stack.

⭐⭐⭐

On-Demand Pricing

IaaS pricing where users pay per hour with no long-term commitment. Maximum flexibility. Best when requirements are unknown.

⭐⭐

Auctioned/Spot Pricing

Users bid for compute resources. Price is market-driven by supply/demand. Best for non-production, non-urgent workloads. AWS Spot Instances use this model.

⭐⭐⭐

Least Connections (Load Balancing)

Load balancing approach that routes new requests to the server with fewest active connections. Ensures more equal load distribution.

⭐⭐

Dell Boomi

Leader in cloud data integration under PaaS. Integrates 40+ technologies and protocols including Oracle, SAP, Salesforce, Hadoop, QuickBooks.

⭐⭐

Auto-Scaling

IaaS feature that automatically scales resources up (scale-out) or down (scale-in) based on workload demands without manual intervention.

🧠 Memory Tricks
🚗 IaaS = Renting a Car

You rent the car (infrastructure). You drive it (manage apps). You fuel it (pay for use). Car company maintains it (CSP manages hardware).

🏗️ PaaS = Renting an Apartment

Building + walls provided (platform). You furnish it (your app). You don't manage plumbing or electricity (OS/middleware managed by provider).

💰 3 IaaS Pricing — "OPA"

On-demand (flexible, most expensive) | Prepaid/Reserved (commit ahead, cheaper) | Auctioned/Spot (bid, cheapest but not immediate)

Think: "Order Pizza Ahead" — ordering on the spot is priciest, planning ahead saves money, bidding on leftover pizza is cheapest!

⚖️ Load Balancing Approaches — "RWD LF"

Round Robin | Weighted Round Robin | Dynamic Round Robin | Least Connections | Fastest Server
