MODULE 1 — CHAPTER 1

☁️ Introduction to Cloud Computing

NIST definition, 5 essential characteristics, related technologies, SOA, distributed systems

⏱ ~45 min read 📊 15-20% weightage ⭐ High priority
🎯 Learning Objectives: After this chapter, you'll know the NIST definition, 5 essential characteristics, cloud history, and how cloud relates to distributed systems, virtualization, grid computing, and SOA.
1.2 What is Cloud Computing?

Simple analogy: Think of cloud computing like electricity. You don't build your own power plant — you plug into the grid and pay for what you use. Cloud computing is the same idea but for IT resources (servers, storage, software).

  • IBM formally introduced cloud computing in late 2007
  • Cloud = delivering hosted services over the internet, on-demand, with pay-as-you-go pricing
  • Cloud is also defined as a large-scale distributed computing paradigm where services are provided via virtual machines
⭐⭐⭐ The NIST Definition (Must Memorize):
"Cloud computing is a model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources (networks, servers, storage, applications, and services) that can be rapidly provisioned and released with minimal management effort or service provider interaction."
1.3 The 5 Essential Characteristics (NIST)

These are the MOST important 5 points in this chapter. They appear in every NPTEL exam.

1. 🖱 On-Demand Self-Service

Users can get computing resources automatically without human interaction from the provider. Like an ATM — you get cash without talking to a bank teller.

2. 🌐 Broad Network Access

Cloud services are accessible over standard networks (internet) through heterogeneous devices — laptops, phones, tablets. Location independent!

3. 🏊 Resource Pooling

Provider's resources are pooled and shared among multiple users (multi-tenancy). Resources are dynamically assigned based on demand. Users typically don't control or know the exact location of resources.

4. ↕️ Rapid Elasticity

Resources can be scaled up or down quickly. To users, the cloud appears to have infinite capacity. Think of a rubber band — stretch it when needed, shrink it back when not.

5. 📊 Measured Service

Cloud systems automatically control and optimize resource use. Usage is monitored, measured, and billed transparently (pay-as-you-go). Like your electricity meter.

Additional Cloud Characteristics
  • Multi-tenancy: Multiple users share the same physical resources (kept isolated and secure)
  • Reliability: Commitment to zero downtime — SLA guarantees
  • Scalability: Increase or decrease resources as needed
  • Reduced time to market: Small orgs and startups can launch without large upfront investment
1.4 Related Technologies & Cloud Predecessors
🖥 Mainframe Computing ▼
  • Large computing facilities with multiple processing units
  • Presented as a single entity to users
  • Specialized for massive data processing (banking, ERP, online transactions)
  • Batch processing was the main application
  • Very reliable — could tolerate failure transparently
  • Popularity decreased but evolved versions still used today
🔗 Cluster Computing ▼
  • Multiple computers combined forming one cluster
  • Low-cost alternative to mainframes and supercomputers
  • From end-user perspective, feels like a standalone system
  • LAN acts as the connection unit with high-bandwidth network
  • Used for: Petroleum Reservoir Simulation, Google Search Engine, Weather Forecasting
  • Easily extended if more computational power needed
⚡ Grid Computing ▼
  • Appeared in early 90s as evolution of cluster computing
  • Cloud computing is considered the successor of grid computing
  • Key difference: Grid is application-oriented, Cloud is service-oriented
  • Resources are distributed in grid, centrally managed in cloud
  • Both use network technology and support multi-tasking
🔲 Virtualization ▼
  • Separation of server software from hardware
  • Core technology enabling cloud computing
  • Virtual server = OS + applications + storage — all in software
  • Multiple OS stacks coexist on same hardware, isolated from each other
  • Examples: Amazon EC2, VMware vCloud, IBM Bloomix
🔧 Service Oriented Computing (SOC) & SOA ▼
  • SOC = computing strategy using services as fundamental elements
  • SOA = Service Oriented Architecture — reorganizes software into interacting services
  • SOA is the core reference model for cloud computing
  • Services must be: technologically neutral, loosely coupled, support location transparency
  • SOC introduced two key concepts for cloud: Quality of Service (QoS) and SaaS
  • Cloud vs SOA: In cloud you pay for infrastructure outcome; in SOA you pay for technology behind service
Real World Impact
  • Large enterprises: Offload activities to cloud to reduce costs
  • Startups: Launch business ideas quickly without upfront investment
  • Developers: Don't worry about infrastructure complexity
  • End users: Access services from anywhere on any device
⚠️ Common Mistake: Students confuse grid computing and cloud computing. Remember: Grid = application-oriented, Cloud = service-oriented. Grid has distributed resource management, Cloud has centralized resource management.
📋 Key Definitions
Cloud Computing (NIST)

A model for enabling ubiquitous, convenient, on-demand network access to a shared pool of configurable computing resources that can be rapidly provisioned and released with minimal management effort.

⭐⭐⭐ Must memorize — appears in exam definition questions

On-Demand Self-Service

The ability to provision computing capabilities automatically without requiring human interaction with the service provider.

⭐⭐⭐ NIST Characteristic 1

Broad Network Access

Capabilities are available over the network and accessed through standard mechanisms that promote use by heterogeneous thin or thick client platforms.

⭐⭐ NIST Characteristic 2

Resource Pooling

Provider's computing resources are pooled to serve multiple consumers using a multi-tenant model, with different physical and virtual resources dynamically assigned and reassigned according to consumer demand.

⭐⭐⭐ NIST Characteristic 3

Rapid Elasticity

Capabilities can be elastically provisioned and released to scale rapidly outward and inward commensurate with demand. To the consumer, available resources appear infinite.

⭐⭐⭐ NIST Characteristic 4

Measured Service

Cloud systems automatically control and optimize resource use by leveraging a metering capability. Resource usage can be monitored, controlled, and reported (pay-as-you-go).

⭐⭐⭐ NIST Characteristic 5

Virtualization

Separation of server software from hardware. Allows creation of virtual computing environments that simulate hardware interfaces. Core enabling technology for cloud computing.

⭐⭐⭐ Core cloud technology

SOA (Service Oriented Architecture)

Way of reorganizing software applications and infrastructure into a set of interacting services. Services are technologically neutral, loosely coupled, and support location transparency.

⭐⭐ Core reference model for cloud

Multi-Tenancy

Multiple users (tenants) sharing the same computing resources (memory, CPU, etc.) while being kept transparent and isolated from each other by the cloud provider.

⭐⭐ Fundamental cloud concept

🌐 Grid vs Cloud — Quick Trick

Grid = Goes Distributed, for big Applications

Cloud = Centralised, for Services

Grid → Distributed → Application-oriented | Cloud → Centralized → Service-oriented

📚 3 Predecessors of Cloud — "MCG"

Mainframe → Cluster → Grid → Cloud

Remember as "My Computer Got Cloud" — the evolution story!

💡 SOA vs Cloud — "Pay Different Things"

In SOA you pay for technology behind the service

In Cloud you pay for infrastructure/platform outcome

SOA services are software components; Cloud services can go up and down the stack!

🔑 Rapid Elasticity = "Rubber Band"

Stretch when you need more resources, snap back when done. And to users — it looks like infinite resources!
