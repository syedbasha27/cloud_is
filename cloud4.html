MODULE 1 — CHAPTER 4

🏢 Cloud Types & Deployment Models

Public, Private, Hybrid, Community Cloud — Components, Implementation, Pros & Cons

⏱ ~45 min 📊 20% weightage ⭐⭐⭐ Very High Priority
🎯 Learning Objectives: Know the 4 cloud deployment types (Public/Private/Hybrid/Community), their advantages/disadvantages, private cloud components, and implementation steps.
4.2 The 4 Cloud Deployment Types

🌐 Public Cloud

What: Infrastructure available to the general public over the internet. Managed by CSP.

Key feature: Multi-tenancy (multiple users share resources)

Examples: Amazon EC2, Microsoft Azure, Google Cloud Platform, IBM SmartCloud, Windows Azure, Gmail

Use cases: Storage, testing, application hosting, development environments

✅ Advantages:

  • No responsibility for buying/maintaining hardware
  • Easy to integrate — highly flexible
  • Maintained by CSP
  • Lower cost than private/hybrid
  • Location independent
  • Nearly unlimited scalability
  • Highly reliable (vast network of servers)

❌ Disadvantages:

  • Less security (resources shared publicly)
  • Internet bandwidth is a bottleneck
  • Client has no control of data (lesser autonomy)
  • Less customization
  • May not meet industry standards/government policies
  • Sudden changes by CSP can have dire impacts

🔒 Private Cloud

What: Cloud infrastructure dedicated exclusively to ONE organization. Also called "corporate" or "internal" cloud.

Key feature: Highest security and control

Open-source tools: OpenStack, Open Nebula, Eucalyptus

Providers: IBM, VMware, HPE, Oracle, Dell EMC

Users: Government bodies, financial organizations, large corporates with business-critical data

Note: Private cloud does NOT require being online always

✅ Advantages:

  • High level of security and privacy
  • Better autonomy — complete control over infrastructure
  • Can meet strict security, compliance, or legal requirements
  • Quick on-demand IT resource allocation
  • No risk of sudden CSP changes

❌ Disadvantages:

  • Requires expert IT personnel to manage
  • Limited to within the organization
  • High investment for deployment and maintenance
  • New operational processes required
  • Less scalable than public cloud

🔀 Hybrid Cloud

What: Combination of public + private cloud connected via LAN/WAN/VPN

Best when: (1) Data can't go to cloud due to legal reasons, (2) Want to keep old hardware running for legacy apps

Example scenario: App deployed on public cloud (scalability) + Database on private cloud (security)

Examples: Google Suite (Gmail, Google Apps, Drive), Office 365

✅ Advantages:

  • Flexibility + scalability vs on-premises
  • Reduced time to market
  • Operational flexibility (critical workloads on private, testing on public)
  • Flexible public resources + secure private resources

❌ Disadvantages:

  • Security compromised compared to pure private
  • Reliability depends on CSP
  • Complex to manage (heterogeneous models)

👥 Community Cloud

What: Shared infrastructure among organizations with shared concerns/tasks

Managed by: One/more organizations in community, third party, or combination

Sectors: Media, Healthcare, Energy, Scientific Research

Security: More than public, less than private. Shared compliance requirements.

4.3 Private Cloud Components

Generic term for components: "Cloud Infrastructure"

  • Hardware: Routers, switches, load balancers, firewall, storage arrays, servers, backup devices
  • Virtualization: Hypervisor sits on hardware, abstracts resources (compute, storage, memory)
  • Storage: Storage management for backup, data removal, reliability. Storage virtualization abstracts physical storage.
  • Network: Physical switches, routers, cables + virtual networks (VLANs). Static/dynamic IP assignment.
4.4 Private Cloud Implementation — Key Building Blocks
  • Virtualization (combines physical resources into shared pools)
  • Service catalog
  • Self-service portal
  • Support for heterogeneous systems
  • Resource and workload manager
  • Metering software and pay-per-use billing

Implementation Steps:

  1. Hardware Setup: Select optimal hardware for application demands
  2. Install management software (the "stack"): OpenStack, VMware vSphere, or CloudStack
  3. Configure management software controller: Interface for creating and managing VMs. Can be on separate server for redundancy.
  4. Create VM templates and deploy server images
  5. Create, License and Test VMs: Final troubleshooting and testing
⭐ Key management software: OpenStack, VMware vSphere, CloudStack. Hypervisors: ESXi, Xen, KVM.
📋 Key Definitions
Public Cloud

Cloud infrastructure open to the general public. Multi-tenant. Managed and operated by CSP. Pay-per-use. Examples: AWS, Azure, GCP.

⭐⭐⭐

Private Cloud

Cloud infrastructure dedicated exclusively to one organization. Also called corporate or internal cloud. Highest security and control. Tools: OpenStack, Eucalyptus.

⭐⭐⭐

Hybrid Cloud

Combination of public and private cloud environments connected via LAN/WAN/VPN. Best of both worlds.

⭐⭐⭐

Community Cloud

Shared infrastructure for organizations with shared concerns or tasks. More security than public, less than private. Used in healthcare, research, media.

⭐⭐

Hypervisor

Software that sits on top of physical hardware and abstracts machine resources to create virtual machines. Also called VMM (Virtual Machine Monitor). Examples: ESXi, Xen, KVM.

⭐⭐⭐

OpenStack

Open-source cloud platform used to implement private clouds using dedicated resources. One of the most common private cloud management tools.

⭐⭐

VLAN (Virtual Local Area Network)

Virtual network created on top of physical network resources in cloud infrastructure. Used to logically segment network traffic.

☁️ 4 Cloud Types — "PPHC"

Public | Private | Hybrid | Community

Remember as: "People Prefer Hybrid Computing" or "Pub, Priv, Hybrid, Community"

🔒 Security Order (Low→High)

Public (least) → Community → Hybrid → Private (most)

Think: The more public → less secure. The more private → more secure!

💰 Cost Order (Low→High)

Public (cheapest) → Hybrid → Private (most expensive)

🔧 Private Cloud Tools — "VOSS"

VMware vSphere | OpenStack | SCloudStack | (Eucalyptus)

Think: "VOS — Very Open Software"

🖥 Hypervisors — "EXK"
