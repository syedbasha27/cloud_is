MODULE 2 — CHAPTER 6

📱 SaaS & DBaaS

Software as a Service delivery models, features, limitations, Database as a Service architecture

⏱ ~45 min 📊 15% weightage
🎯 Learning Objectives: Understand SaaS characteristics, delivery models, benefits, limitations; understand DBaaS and its architecture components; know how to select a DBaaS vendor.
2.2 Software as a Service (SaaS)

SaaS = software deployed on a hosted service accessible over the internet. Analogy: Instead of buying Microsoft Office, you use Google Docs — no install needed, always updated, accessible anywhere.

Two events that increased SaaS popularity:

  1. The internet became a commercial platform
  2. Installing, running, upgrading and managing software became costly and complex

Most common SaaS categories:

  • Billing and Inventory Systems
  • CRM (Customer Relationship Management) software
  • Help desk applications
  • HR management applications
  • Security-related software
  • Social service software

Desired Characteristics of SaaS:

  • Generalized (to appeal to many customers)
  • Modular and service-oriented
  • Built-in measuring and monitoring (usage-based billing)
  • Sophisticated navigation
  • Built-in billing service
  • Data integrity, reliability, and security
  • Constant faster releases without affecting existing customers

Key Features of SaaS:

  • Available over the Internet
  • Usage or subscription-based licensing
  • No maintenance at end-user side
  • Available on-demand, scalable up/down
  • Vendor maintains the software
  • Automatically upgraded and updated
  • All users run the same version
SaaS Benefits
  • Minimal Software Tools: No installation, minimal config, less distribution cost
  • Centralized Data Management: Provider handles reliability, security, encryption
  • Multitenant Solutions: Multiple users share single instance, virtual isolation, customizable
  • Efficient License Usage: Single license for multiple computers, no license servers needed
  • Reduced Cost: Multi-tenant architecture reduces cost
SaaS Delivery Models (3 Categories)

1. 📦 Packaged Software Services

Biggest area in SaaS market. Focus on specific business processes. Examples: NetSuite (CRM/ERP), RightNow (CRM), Constant Contact (marketing automation), Intuit (accounting).

2. 🤝 Collaborative Software Services

For geographically distributed teams. Web conferencing, document collaboration, project planning, instant messaging. Examples: Microsoft Live, Google Apps, Zoho, Cisco WebEx.

3. 🔧 Enabling & Management Services

Support development and deployment of SaaS apps: Testing-as-a-service (HP, IBM, Sogeti), Monitoring-as-a-service, Development-as-a-service, Security-as-a-service (Symantec, McAfee, CA, Kaspersky), Compliance & Governance services.

SaaS Limitations
  • Not suitable for real-time systems: Network delay makes SaaS unsuitable
  • Internet dependency: Must have internet to use the app
  • Data security and compliance: Data stored on cloud, privacy concerns
  • Lack of interoperability and vendor lock-in: Flexibility only if vendor supports it
  • Less customization: Generalized design may not suit specific needs
  • Limited range: Not all desktop apps have SaaS equivalents yet
2.3 Database as a Service (DBaaS)

DBaaS = IT providers deliver database functionality as a service. Customers focus on application development while provider manages all database infrastructure.

What DBaaS relieves you from:

  • Costly hardware/software purchases
  • Compatibility checks
  • Dependency resolutions
  • Backup and Cron jobs
  • Hardware or OS failures
  • System upgrades
  • Management of Firewall rules

3 DBaaS Service Categories:

  • Provisioning Services: Request new DBs, choose options (partitioning, security, clusters), add/remove computing resources, backup capability
  • Administrative Services: On-demand restores, database cloning, performance monitoring and alerting
  • Reporting Services: Performance management via GUI, resource consumption reports, SLA compliance tracking

7 Criteria for Selecting DBaaS Vendor:

  1. Location of Database (close to app server = less latency + security)
  2. Fault Tolerance, Availability and Redundancy (multi-node clusters, SLA)
  3. Data durability via backup and restore
  4. Performance Monitoring, analysis and Alerts
  5. Performance and Scaling (vertical = add RAM/CPU, horizontal = add nodes)
  6. Security (encryption, SSL, authentication, firewall, audit)
  7. Support (included in price? SLA for support?)
DBaaS Conceptual Architecture (6 Components)
  • DBaaS Development: Define service offerings, manage service catalogue
  • DBaaS Management: Subscriber Services + Service Management
  • DBaaS Service Deployment: Physical resources and configuration
  • Infrastructure: Physical servers, networking, software
  • IT Service Management: Service definition, design, operational practices, policies
  • Subscriber Interfaces: Systems/procedures for interacting with DBaaS
📋 Key Definitions
SaaS (Software as a Service)

Cloud delivery model where software is provided online on-demand. No installation required. Vendor maintains and updates software. All users run the same version.

⭐⭐⭐

DBaaS (Database as a Service)

Architectural approach where IT providers deliver database functionality as a service. Provider manages all DBA tasks (backup, scaling, security, monitoring).

⭐⭐⭐

Multi-tenancy (SaaS)

Multiple customers share a single instance of data and application via virtual isolation. Each customer can customize without affecting core functionality.

⭐⭐

Vertical Scaling (DBaaS)

Adding more resources (RAM, CPU, Disks) to existing database servers as data volume grows.

⭐⭐

Horizontal Scaling (DBaaS)

Adding more nodes to the database system to handle growth in data volume. Distributes load across multiple servers.

⭐⭐

↕️ Vertical vs Horizontal Scaling

Vertical = Taller building (add RAM/CPU to same server — "scaling UP")
Horizontal = More buildings (add more servers — "scaling OUT")
